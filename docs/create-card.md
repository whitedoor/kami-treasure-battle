# カードの生成ロジック

* ブラウザ上でカメラを起動して、レシートを撮影する
* レシートの写真をGCPサービスを用いて以下の情報を抽出する
    * 商品名
    * 金額
* 一度、データベースに保存する
* その後、保存した商品名、金額をプロンプトとしてGPCサービスに送り、以下を生成する
    * テキストデータ
        * カード名：魔法っぽい名前で生成する
        * 攻撃力：20, 30, 40, 50のどれかで、数字が大きくなればなるほど出力される確率を下げたい
        * じゃんけんは、商品名からそれっぽい種類（ぐー、ちょき、ぱー）にする
    * 画像データ：魔法を唱えている感のある画像。正方形にする
* テキストデータをデータベースに保存し、生成したユーザーと紐づける
* 画像データもGCPサービスに保存し、テキストデータと同時に画像のファイル名もidと紐づけておく
* 攻撃力に応じてカードのレアリティも決めて、テーブルに保存する
    * 20：ブロンズ
    * 30：シルバー
    * 40：ゴールド
    * 50：レジェンド
